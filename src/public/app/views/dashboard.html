QueryPro Demo

QueryPro is a full featured database query analyzer. When creating complex data driven software systems it becomes difficult to track exactly which queries are being ran when, and how applications, or components of the system, are effecting database server performance. While many database servers have logs which can track query data, they are rarely run in production environments due to the performance overhead. There are also paid enterprise solutions available, but they typically only monitor the server level and are not cost effictive for smaller organizations. What is unique about QueryPro, is that QueryPro offers both server and application level moniotoring. This allows for tracking both server performance and application thorough put allowing for in depth analysis of the performance relationship between a database server and various applications.


How it Works

QueryPro is a collection of Docker containers, which work together to collect, aggregate, and display data on both server and application performance. By defining the QueryPro architecture in Docker, installation is incredibly easy, simply install Docker on a server droplet and run the containers. Once started, the containers will immediately begin collecting server data on predefined servers and start storing the data which will immediately be available via the droplet's IP address using the QueryPro UI. To add application data, simply download one of the QueryPro libraries and embed a few lines of code within your application's ORM. All QueryPro interactions are done via UDP (opposed to TCP), adding no measuarable overhead**. There is no limit on the number of applications, which can be unique software systems or distinct components of a single system.


Current Features

  • Application Monitoring - See all stats per application (drop down top right)
  • Query Frequency - Monitor how many queries are being ran per hour/day and what are the slowest query times for that period
  • Query Distribution - See a break down of CRUD queries per day
  • Slow Queries - List the ten slowest queries per day for the last week

Future Features

  • Deviations - Charts for monitoring deviations of both query frequency and efficiency
  • Alerts - Ability to set alerts i.e. 10% queries > 500ms etc.
  • Monitor - Have a monitoring system built on collectD for capturing server performance stats
